# P1 开发计划（需求文档）

## Context

P0 已稳定运行，补丁改进已完成（AI 角色从主笔改为参考建议模式、水事件过滤、标签体系锁定三个固定标签）。P1 新增 6 个功能，不修改 P0 已有的数据库表结构。

## 实施顺序

```
功能1（数据导出）→ 功能2（迭代链可视化）→ 功能3（统计面板）→ 功能4（全文搜索）→ 功能5（离线写入）→ 功能6（原生APP + 溯源视图双端布局）
```

---

## 功能 1：数据导出（JSON / CSV）

用户可以在设置页导出全部数据，支持 JSON 和 CSV 两种格式。CSV 导出为 ZIP 包，包含日记、复盘、心得三个文件。中文编码不能乱码。

---

## 功能 2：认知迭代链可视化

心得详情页和数据库浏览页中，如果某条心得存在迭代链（related_insight_id），以垂直时间线形式展示完整迭代历史。每个节点显示版本号、日期、内容摘要，当前版本高亮，点击可跳转到对应心得。

---

## 功能 3：统计面板

新增统计页面，展示用户的复盘和心得产出数据。

内容包括：
- 周/月切换
- 数字概览：复盘总数、心得总数、成功率
- 趋势图：每日复盘/心得产出数量
- 分布图：成功/失败/认知迭代的比例

导航栏增加"统计"入口。

---

## 功能 4：全文搜索

用户可以通过关键词搜索日记、复盘、心得的内容。

要求：
- 支持中文搜索
- 搜索结果按类型分类（日记/复盘/心得），可切换筛选
- 搜索结果展示匹配片段并高亮关键词
- 点击搜索结果跳转到对应详情页

导航栏增加"搜索"入口。

---

## 功能 5：离线写入

手机端断网时仍可写入碎片。碎片暂存在本地，联网后自动同步到服务器。

要求：
- 断网时写入的碎片显示"待同步"标记
- 联网后自动逐条同步，同步成功后标记消失
- 同步失败的碎片保留，等下次联网重试
- 页面上显示在线/离线状态指示器

---

## 功能 6：原生 APP + 溯源视图双端布局

### 原生 APP

使用 React Native 开发移动端 APP，复用现有后端 API，不需要改后端。

APP 需要包含的页面：
- 写入页（随手记 + 写日记）
- 日记页（列表 + 详情 + 确认）
- 复盘页（左栏AI建议 + 右栏用户手写，移动端改为上下布局）
- 心得页（同上）
- 数据库浏览页（三个Tab + 标签筛选）
- 溯源视图（移动端专用布局，见下方）
- 设置页

### 溯源视图双端布局

**桌面端（Web）：** 横向树形图，从左到右展开——碎片（带时间戳）→ 日记 → 事件列表 → 复盘 → 心得。点击任意节点可展开详情。

**移动端（APP）：** 纵向瀑布流，从上到下展开——碎片列表（可折叠）→ 日记卡片 → 事件卡片列表。每个事件卡片带标签颜色（成功绿、失败红、中性灰），点击跳转复盘详情页。

---

## 验证方式

1. 数据导出：导出 JSON 和 CSV，检查内容完整性和中文编码
2. 迭代链：创建多个版本的心得，验证时间线展示和跳转
3. 统计面板：写入多天数据，检查图表数据准确性和周/月切换
4. 全文搜索：搜索中文关键词，验证结果和高亮
5. 离线写入：断网写碎片 → 联网自动同步 → 检查数据
6. 原生APP：全流程测试（写入→日记→复盘→心得→数据库浏览→溯源）
7. 溯源视图：桌面端验证横向树形图，移动端验证纵向瀑布流
